GDI / GDC
Графический интерфейс (контекст) устройства
Договорённость / набор инструментов для "рисования" на устройстве

Контекст устройства (граф. контекст) - адаптер, промежуточное звено, реализующее интерфейс - переводящее граф. команды
	в реальное граф. отображение.

Для рисования на окне:
1. Получаем контекст устройств (окна)
	HDC dc = GetDC(hWnd);
2. Создаем инструмент рисования
	HPEN pen = CreatePen(PS_SOLID, 2, RGB(0,0,0));
3. Устанавливаем этот инструмент. Принцип "выталкивания" - 
	результатом установки есть деструктор пред. инструмента
	HPEN savedPen = (HPEN)SelectObject(dc, pen);
4. Используем
	LineTo(dc, 100, 100);
5. Восстанавливаем инструмент, который мы "вытолкнули"
	SelectObject(dc, savedPen);
6. Освобождаем инструмент
	DeleteObject(pen);
7. Освобождаем контекст устройства 
	ReleaseDC(hWnd, dc);

1, 7 - контекст устройств регистрируется в системе (есть ресурсом ОС). При его создании система возвращает дескриптор по которому
	   его нужно позже удалить. Если ресурс не удаляется, то происходит утечка ресурсов
2, 6 - суть та же. Ресурсы рисования - перья (Pen) и кисти (Brush). Перья рисуют линии, кисти закрашивают(заливают) области.
3, 5 - контекст устройства может использоваться для разных задач (разными объектами, разными сообщениями).
	   Если оставить установленный ресурс, не восстановив стартый, то это может повлиять на работу других процессов. 
4 - использование интерфейсных функций - стандартных для разных языковв / устройств.
	MoveToEx - переход в точку (без рисования)
	LineTo - переход в точку (с рисованием)

Захват мыши
SetCapture(hWnd) - все события мыши будет получать окно hWnd, даже если курсор мыши выйдет за него
ReleaseCapture() - снимает захват
